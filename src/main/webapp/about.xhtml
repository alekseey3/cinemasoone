О проекте CinemaSoon

CinemaSoon — учебное веб-приложение на платформе Jakarta EE для учёта ожидаемых фильмов. Приложение позволяет добавлять новые фильмы, изменять уже существующие записи, удалять их, а также ставить лайки самым ожидаемым премьерам.

Реализованный функционал

  • добавление фильма с указанием названия, даты выхода, жанра и описания;
  • просмотр общего списка ожидаемых фильмов;
  • редактирование данных о фильме;
  • удаление фильма из списка;
  • лайки для фильмов и формирование «Топа ожиданий» по количеству лайков;
  • просмотр фильмов по жанрам через отдельный раздел.

Используемые технологии и инструменты

  • Jakarta EE: EJB (бизнес-логика), JPA (доступ к данным), JSF/Facelets (представление);
  • Сервер приложений: Eclipse GlassFish;
  • СУБД: PostgreSQL;
  • Инструменты разработки: IntelliJ IDEA, pgAdmin;
  • Деплой: развёртывание приложения выполняется через консоль администратора GlassFish, без использования средств деплоя IDE.

Архитектура приложения

Приложение построено по типовой трёхслойной архитектуре Jakarta EE:

  • Слой данных (Entity): сущности User и Movie, маппинг на таблицы users и movies в PostgreSQL. В таблице movies хранится информация о фильмах (название, дата выхода, жанр, описание, количество лайков, дата создания и связь с пользователем).
  • Бизнес-логика (EJB): сессионный компонент MovieService, отвечающий за операции с фильмами: создание, чтение, обновление, удаление, фильтрацию по жанру и обработку лайков.
  • Слой представления (JSF): managed-bean MovieBean и набор страниц Facelets:
    • index.xhtml — основная работа со списком фильмов (CRUD и лайки);
    • genres.xhtml — просмотр фильмов по выбранному жанру;
    • top.xhtml — раздел «Топ ожиданий» по количеству лайков;
    • about.xhtml — описание проекта и его архитектуры.

Работа с базой данных

Подключение к PostgreSQL выполняется через пул соединений GlassFish (PostgresPool) и JDBC-ресурс jdbc/CinemaSoonDS. Этот ресурс указывается в конфигурации JPA как jta-data-source в единице персистентности CinemaSoonPU. Благодаря этому все операции с сущностями Movie и User выполняются через стандартный механизм JPA.

В итоге проект демонстрирует полноценную трёхслойную архитектуру: слой данных (PostgreSQL + JPA), бизнес-слой (EJB) и слой представления (JSF), а также полную цепочку — от формы на веб-странице до записи в базе данных.
